                    │

                    │
                    ▼

═════════════════════════════════

    POST /api/ticket with {ticket_text}
            │
            ▼
    ┌─────────────────────────────────────────┐
    │   handle_process_ticket()               │
    │   ├─ Basic validation (length, etc)     │
    │   ├─ PIIDetector.find_pii()             │
    │   └─ Checks for all 8 PII types        │
    └──────────────┬──────────────────────────┘
                   │
        ┌──────────┴──────────┐
        │                     │
    No PII Found          PII DETECTED
        │                     │
        │                 Return 400 Error:
        │                 {
    Continue                "warning": "...",
        │               "detected_pii_types": [
        ▼                   "emails",
                            "phone_numbers"
    Proceed to              ],
    Service Layer       "examples": {...}
                        }


LAYER 3: SERVICE PROCESSING (DEFENSIVE MASKING)
═══════════════════════════════════════════════

    process_support_ticket(ticket_text)
            │
            ▼
    ┌──────────────────────────────────────────┐
    │   PIIMasker.mask_pii(ticket_text)        │
    │   ├─ Masks: emails     → [EMAIL_MASKED]  │
    │   ├─ Masks: phones     → [PHONE_MASKED]  │
    │   ├─ Masks: SSN        → [SSN_MASKED]    │
    │   ├─ Masks: CC         → [CC_MASKED]     │
    │   ├─ Masks: Account ID → [ACCTID_MASKED] │
    │   ├─ Masks: IP         → [IP_MASKED]     │
    │   └─ Returns: (masked_text, detected)    │
    └──────────────┬──────────────────────────┘
                   │
                   ▼
    ┌──────────────────────────────────────────┐
    │   LLM Call 1: Ticket Analysis            │
    │   (Uses MASKED text)                     │
    │   ├─ Analyzes category, urgency, etc    │
    │   └─ Returns JSON analysis              │
    └──────────────┬──────────────────────────┘
                   │
                   ▼
    ┌──────────────────────────────────────────┐
    │   LLM Call 2: Guidance Generation        │
    │   (Uses MASKED text)                     │
    │   ├─ Creates troubleshooting steps       │
    │   └─ Returns guidance text               │
    └──────────────┬──────────────────────────┘
                   │
                   ▼
    ┌──────────────────────────────────────────┐
    │   LLM Call 3: Email Generation           │
    │   (Uses MASKED text + analysis)          │
    │   ├─ Creates customer email response     │
    │   └─ Returns email draft                 │
    └──────────────┬──────────────────────────┘
                   │
                   ▼
            Return Results
            (With masked data)
